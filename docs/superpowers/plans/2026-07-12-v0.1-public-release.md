# nowdocs v0.1 Public Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish nowdocs as a complete public v0.1 release: a verified five-platform binary, a crates.io package usable by cargo-binstall, a working Homebrew tap, and three installable, integrity-checked canonical registry docsets.

**Architecture:** Treat the release as one supply chain with four independently verifiable links: source package -> platform artifacts -> installation clients -> registry content. Keep the product's current local/MCP architecture unchanged. Registry contributors continue to submit text-only share bundles; the release pipeline builds the trusted `.lance` artifacts, signs nothing, and publishes SHA-256 checksums.

**Tech Stack:** Rust 2021 / Cargo, GitHub Actions, GitHub Releases, crates.io, cargo-binstall, Homebrew, LanceDB, Candle/Jina, `gh`, `curl`, `tar`.

## Release Decisions Locked by This Plan

- This is a **complete public release**, not a source-only preview. `nowdocs install`, `cargo binstall nowdocs`, and Homebrew must work before announcement.
- Do not move the existing remote `v0.1.0` tag again. It is 144 commits behind `main` and already has failed release history. Publish the first immutable release as **v0.1.1**; Cargo package version is `0.1.1`.
- Registry repository names are `nextjs-docs`, `react-docs`, and `vue-docs`; public CLI docset names remain `nextjs`, `react`, and `vue`.
- Registry releases are trusted CI-built archives containing a prebuilt table named after the public docset (`nextjs.lance/`, `react.lance/`, or `vue.lance/`) plus `manifest.json`, `LICENSE`, and `NOTICES`. Contributor share bundles remain text-only and never contain vectors.
- Release archives use `.tar.gz` for every target. cargo-binstall metadata must use a literal `.tar.gz` filename and point to the executable within the archive.
- The public support floor is Rust **1.97.0**, because it is the only compiler version verified by this release plan. Document a lower MSRV only after it receives its own CI job.
- No `git push`, GitHub Release creation, crates.io publication, registry repository mutation, or GitHub security-setting mutation is performed until the owner explicitly authorizes that external step.

## Global Constraints

- MCP stays stdio NDJSON at protocol `2025-11-25`; `serve` remains argumentless and binds no network port.
- v1 remains English-first, uses exact/flat vector search, Candle + `jinaai/jina-embeddings-v2-small-en`, and keeps vectors at 512 dimensions/f16 storage.
- Model provenance and model SHA validation remain mandatory. Registry package SHA-256 and manifest docset identity remain mandatory.
- Canonical registry licenses are only MIT, Apache-2.0, or CC-BY-4.0; CC-BY packages require attribution and `LICENSE`/`NOTICES`.
- Do not weaken the registry URL allowlist, test-only `file://` gate, prompt-injection sanitizer, or text-only contributor sharing rule.
- Work on code in an isolated feature worktree. Documentation-only plan updates may be committed on `main`; do not touch existing untracked files.
- Every implementation task follows TDD, ends with a focused English conventional commit, and runs `cargo fmt --check`, `cargo clippy --locked --all-targets -- -D warnings`, and the task-specific tests.
