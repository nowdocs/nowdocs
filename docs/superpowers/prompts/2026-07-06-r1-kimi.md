## Task

Implement R1: error taxonomy + registry archive validation foundation.

## Scope

Primary files:

- `src/registry.rs`
- `tests/registry_tests.rs`

Allowed if needed:

- `src/errors.rs` or `src/diagnostics.rs`
- `src/lib.rs`
- `tests/diagnostics_tests.rs`

Do not touch:

- MCP transport or protocol.
- embedder model/spec fields.
- cache layout semantics beyond current install validation.
- staging/rollback logic; that is R2.
- `doctor`, `smoke`, cache CLI, registry discovery; those are later stages.

## Requirements

1. Add a minimal internal error representation for user-facing failures.
   - Include code/category/message/hint.
   - Keep compatibility with `anyhow::Result` at public boundaries.
   - Display should include the code and next-step hint.

2. Validate registry tar archives before install writes active cache paths.

3. Reject:
   - missing `manifest.json`
   - missing `chunks.jsonl`
   - absolute paths
   - `..` path components
   - duplicate `manifest.json`, `chunks.jsonl`, `LICENSE`, `NOTICES`
   - symlink/hardlink/device/unsupported unsafe entries
   - vector artifacts: `.lance`, `.faiss`, `vectors.*`, `embeddings.*`
   - oversized entries/archives based on named constants

4. Preserve existing valid install behavior and registry URL allowlist behavior.

5. Do not introduce network-dependent tests.

## TDD checklist

Write failing tests first, then implement.

Minimum tests:

- missing manifest rejected
- missing chunks rejected
- path traversal rejected
- absolute path rejected
- duplicate manifest/chunks rejected
- vector artifact rejected
- unsupported tar entry rejected if feasible
- oversized archive/entry rejected
- valid archive still installs
- formatted error includes code and hint

## Verification

Run:

```bash
cargo test registry_tests -- --test-threads=1
cargo test diagnostics_tests -- --test-threads=1  # only if added
cargo fmt --check
git diff --check
```

If `diagnostics_tests.rs` is not added, explicitly say the diagnostics command was not applicable.
